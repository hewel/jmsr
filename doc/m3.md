**Reference: JMSR Design System (Material Design 3)**

We have established a centralized CSS system based on Tailwind CSS and Material Design 3 (Dark Theme).
**Do not** manually build components using raw utility classes (e.g., `bg-primary rounded-3xl`) if a component class exists. Use the predefined classes in `index.css`.

## Typography

Use these helper classes to enforce the M3 type scale.

### Display (Large Headers)
```css
.text-display-large   /* 57px/64px - Hero sections */
.text-display-medium  /* 45px/52px - Major headings */
.text-display-small   /* 36px/44px - Section headings */
```

### Headline (Section Headers)
```css
.text-headline-large  /* 32px/40px - Page titles */
.text-headline-medium /* 28px/36px - Section titles */
.text-headline-small  /* 24px/32px - Subsection titles */
```

### Title (Card/Dialog Titles)
```css
.text-title-large   /* 22px/28px - Dialog titles */
.text-title-medium  /* 16px/24px - Card titles, font-medium */
.text-title-small   /* 14px/20px - List item titles, font-medium */
```

### Body (Reading Text)
```css
.text-body-large   /* 16px/24px - Default text */
.text-body-medium  /* 14px/20px - Secondary text */
.text-body-small   /* 12px/16px - Captions */
```

### Label (Buttons, Captions)
```css
.text-label-large  /* 14px/20px - Button text, font-medium */
.text-label-medium /* 12px/16px - Tabs, badges, font-medium */
.text-label-small  /* 11px/16px - Timestamps, font-medium */
```

## Buttons

### Primary Action (High Emphasis)
```tsx
<button class="btn-primary">Connect</button>
```
- **Usage:** The single most important action on a screen (e.g., "Connect", "Save", "Play").
- **Style:** Filled, `bg-primary`, `text-on-primary`, rounded-full, height 40px.

### Tonal Action (Medium Emphasis)
```tsx
<button class="btn-tonal">Cancel</button>
// or
<button class="btn-secondary">Cancel</button>
```
- **Usage:** Alternative actions, secondary buttons.
- **Style:** `bg-secondary-container`, `text-on-secondary-container`, rounded-full.

### Outlined Action (Low Emphasis)
```tsx
<button class="btn-outlined">Learn More</button>
```
- **Usage:** Less prominent actions that need visibility.
- **Style:** Transparent with `border-outline`, `text-primary`, rounded-full.

### Text Action (Lowest Emphasis)
```tsx
<button class="btn-text">Forgot Password?</button>
```
- **Usage:** Inline actions, links within forms.
- **Style:** No background, `text-primary`, minimal padding.

### Icon Button
```tsx
<button class="btn-icon">
  <RefreshCw size={20} />
</button>
```
- **Usage:** Toolbar buttons, playback controls.
- **Style:** 40x40px, rounded-full, subtle hover state.

## Inputs (Text Fields)

### Filled Input (M3 Standard)
```tsx
<input type="text" class="input-filled" placeholder="Enter value..." />
```
- **Style:** `bg-surface-container-highest`, bottom border only, rounded-t-lg.
- **Height:** 56px (h-14).

### Outlined Input
```tsx
<input type="text" class="input-outlined" placeholder="Enter value..." />
```
- **Style:** Transparent with `border-outline`, rounded-lg.
- **Height:** 56px (h-14).

### Default Input (Legacy)
Standard `input[type="text"]`, `input[type="password"]`, `input[type="url"]` elements automatically receive M3 styling unless given `.no-style` class.

### Input with Label Pattern
```tsx
<div class="flex flex-col gap-1">
  <label class="text-label-medium text-on-surface-variant">Server URL</label>
  <input type="text" class="input-filled" placeholder="http://..." />
</div>
```

## Cards & Surfaces

### Elevated Card
```tsx
<div class="card-elevated">
  <h3 class="text-title-medium">Card Title</h3>
  <p class="text-body-medium text-on-surface-variant">Description</p>
</div>
```
- **Usage:** Floating elements, modals, important content.
- **Style:** `bg-surface-container-low`, shadow-md, hover:shadow-lg.

### Filled Card (Default)
```tsx
<div class="card-filled">...</div>
```
- **Usage:** Setting groups, media items, standard containers.
- **Style:** `bg-surface-container-highest`, no shadow.

### Outlined Card
```tsx
<div class="card-outlined">...</div>
```
- **Usage:** When clear visual separation is needed.
- **Style:** `bg-surface`, `border-outline-variant`.

## Icons

Use **Lucide Solid** icons (from `lucide-solid` package).

```tsx
import { Play, Settings, RefreshCw } from 'lucide-solid';

// Standard size
<Play size={20} />
<Settings size={24} />

// With color
<Play class="text-primary" size={20} />
<Settings class="text-on-surface-variant" size={24} />
```

- **Standard sizes:** 20px (inline), 24px (standalone).
- **Color:** Inherits from parent by default, or use `text-*` classes.

## State Layers

M3 uses opacity-based state layers for interactive feedback:

| State | Opacity | Example |
|-------|---------|---------|
| Hover | 8% | `hover:bg-primary/8` |
| Focus | 12% | `focus:bg-primary/12` |
| Pressed | 12% | `active:bg-primary/12` |
| Disabled | 38% opacity on content | `disabled:text-on-surface/38` |

## Scrollbar Styling

Custom scrollbars are automatically applied:
- Track: `bg-surface-container`
- Thumb: `bg-outline-variant`, rounded
- Thumb hover: `bg-outline`

## Quick Reference

### Page Structure
```tsx
<div class="min-h-screen bg-background p-6">
  <div class="max-w-2xl mx-auto">
    <h1 class="text-headline-large text-on-surface mb-4">Page Title</h1>
    <div class="card-filled mb-6">
      <h2 class="text-title-medium mb-4">Section</h2>
      {/* content */}
    </div>
  </div>
</div>
```

### Form Pattern
```tsx
<form class="space-y-4">
  <div class="flex flex-col gap-1">
    <label class="text-label-medium text-on-surface-variant">Label</label>
    <input type="text" placeholder="..." />
  </div>
  <button type="submit" class="btn-primary w-full">Submit</button>
</form>
```

### Status Indicators
```tsx
// Success
<span class="text-tertiary">Connected</span>

// Error
<span class="text-error">Disconnected</span>

// Neutral
<span class="text-on-surface-variant">Pending</span>
```
